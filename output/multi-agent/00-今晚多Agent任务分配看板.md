# 今晚多Agent任务分配看板（总控）

## 目标（明早可验收）
在不增加成本前提下，搭建可持续的多Agent执行系统，并产出可直接使用的策略、模板、计划与质检框架。

## 角色与任务

### Agent A｜系统与流程总设计
- 交付：`01-多Agent总控SOP.md`
- 职责：角色定义、任务路由、交接协议、4小时汇报节奏、08:00晨间反思格式
- 验收：是否能被直接照抄执行

### Agent B｜运营模板工程
- 交付：`02-任务分配与汇报模板.md`
- 职责：任务看板模板、派单模板、完工汇报模板、风险日志模板
- 验收：是否一键复制即可上线

### Agent C｜内容增长执行
- 交付：`03-7天内容增长执行计划.md`
- 职责：公众号 + 小红书 7天执行计划，按“产出物+验收标准”拆解
- 验收：是否每天都能明确“做什么/做到什么程度”

### Agent D｜KPI与质检
- 交付：`04-KPI与质检框架.md`
- 职责：领先/滞后指标、质检清单、止损规则、晨报评分标准
- 验收：是否可用于明早复盘打分

## 协作规则（今晚）
1. 先产出可用版本，再优化。
2. 不做空泛建议，必须给模板与标准。
3. 默认中文，结论先行，步骤清晰。
4. 高风险事项单独标记，其他默认直接推进。
